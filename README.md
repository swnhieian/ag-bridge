# AG Bridge

`AG Bridge` is a VS Code / Antigravity extension that starts a lightweight local bridge service and exposes Antigravity Language Server through stable HTTP and SSE interfaces for automation, session takeover, frontend integration, and scripting.

`AG Bridge` 是一个面向 VS Code / Antigravity 的扩展。它会启动一个轻量的本地 bridge 服务，把 Antigravity Language Server 通过稳定的 HTTP 和 SSE 接口暴露出来，方便做自动化、会话接管、前端联调和脚本集成。

## Overview

- Starts a local bridge service inside the extension host
- Provides a dashboard for the listening address, port, data directory, and sessions
- Lets you copy the current bridge base URL
- Syncs existing Antigravity sessions and lets you continue sending messages to them
- Streams text output, thinking, command output, approval requests, and full step details through HTTP / SSE
- Supports exporting full session data
- Supports CLI, Python SDK, and external script integration
- Includes auto-approval controls for semi-automated or fully automated workflows

- 在扩展宿主里启动本地 bridge 服务
- 提供状态面板，展示监听地址、端口、数据目录和 session
- 支持复制当前 bridge 的 base URL
- 可以同步 Antigravity 里已有的会话，并继续向这些会话发消息
- 通过 HTTP / SSE 输出文本、thinking、命令输出、审批请求和完整 step 详情
- 支持导出完整 session 数据
- 支持 CLI、Python SDK 和外部脚本接入
- 提供自动审批控制，适合半自动或全自动流程

## Typical Use Cases

- Read AG session streams from your own frontend or backend
- Send messages to AG directly from scripts instead of driving the desktop UI
- Pipe AG output into tests, agents, workflow orchestration, or other tooling
- Access full step data instead of only the final text output

- 从你自己的前端或服务端读取 AG 会话流
- 通过脚本直接向 AG 发消息，而不是操作桌面 UI
- 把 AG 的输出接到测试、代理、工作流编排或其他工具
- 拿到完整 step 数据，而不仅仅是最终文本

扩展默认会自动启动本地 bridge 服务。

## Common API Examples

Check service status:

查看服务状态：

```bash
curl http://127.0.0.1:9464/status
```

Create a session:

创建会话：

```bash
curl -X POST http://127.0.0.1:9464/sessions \
  -H 'Content-Type: application/json' \
  -d '{
    "mode": "connect",
    "workspacePath": "/absolute/path/to/project",
    "model": "claude-4-sonnet"
  }'
```

Send a message:

发送消息：

```bash
curl -X POST http://127.0.0.1:9464/sessions/<sessionId>/messages \
  -H 'Content-Type: application/json' \
  -d '{
    "text": "Please reply with OK only.",
    "model": "auto"
  }'
```

Subscribe to the event stream:

订阅实时事件：

```bash
curl -N http://127.0.0.1:9464/sessions/<sessionId>/stream
```

Sync existing AG sessions:

同步已有 AG 会话：

```bash
curl -X POST http://127.0.0.1:9464/ag-sessions/sync \
  -H 'Content-Type: application/json' \
  -d '{}'
```

## Session Identity

`session.id` is generated by the bridge itself and is used for bridge routing, persistence, and CLI / HTTP operations.

`session.id` 是 bridge 自己生成的，用于 bridge 路由、持久化目录以及 CLI / HTTP 的后续操作。

`cascadeId` is the native Antigravity session identifier returned by AG.

`cascadeId` 是 AG 原生返回的会话标识。

`trajectoryId` may appear in AG session discovery results, but it is not the bridge session id.

`trajectoryId` 可能会出现在 AG session 发现结果里，但它不是 bridge session id。

`POST /sessions` can now accept an optional caller-provided `sessionId`. If omitted, the bridge still generates one automatically.

`POST /sessions` 现在可以接收可选的 `sessionId`。如果不传，bridge 仍然会自动生成。

Existing sessions can be resumed through `POST /sessions/:id/resume`, and the CLI also supports `resume` and `chat --last`.

已有 session 可以通过 `POST /sessions/:id/resume` 恢复，CLI 也支持 `resume` 和 `chat --last`。

## Step Detail Support

If you need full AG step data in a frontend, consume `cascade.step.new` and `cascade.step.updated` directly.

如果你要在前端完整展示 AG 的 step，建议直接消费 `cascade.step.new` 和 `cascade.step.updated`。

The `data.step` field in these events contains full step detail, including summarized fields, the current oneof payload, and protobuf-style raw JSON.

这两个事件里的 `data.step` 会带完整 step 详情，包括摘要字段、当前 oneof payload，以及 protobuf 风格的原始 JSON。

## Model Selection

CLI, HTTP, and the Python SDK all support selecting a model through the `model` field.

CLI、HTTP 和 Python SDK 都支持通过 `model` 字段指定模型。

Common examples:

常见写法：

- `claude-4-sonnet`
- `google-gemini-2-5-pro`
- `auto`
- `recommended`

View the models currently returned by the language server:

查看当前语言服务器实际返回的模型：

```bash
node dist/src/cli.js list-models
```

Resume the most recent session:

恢复最近一个 session：

```bash
node dist/src/cli.js resume --last
```
