## Which One to Read

## 怎么选

Read the build guide if you are packaging the extension, generating VSIX files, producing CLI binaries, or preparing Open VSX releases.

如果你要打包扩展、生成 VSIX、制作 CLI 二进制，或者准备 Open VSX 发布，请看构建文档。

Read the Node CLI guide if you mainly use `node ./bin/ag-bridge.js` or the packaged native CLI for automation, discovery, sessions, chat, streaming, and `--auto-approve`.

如果你主要使用 `node ./bin/ag-bridge.js` 或打包后的原生 CLI 做自动化、discovery、session、chat、stream 和 `--auto-approve`，请看 Node CLI 文档。

Read the Python guide if you want to call the local bridge from Python scripts, backend jobs, or the bundled `ag-bridge-py` command.

如果你想在 Python 脚本、后端任务或 `ag-bridge-py` 命令里调用本地 bridge，请看 Python 文档。

Read the API reference if you want the raw HTTP endpoints, request parameters, and SSE / event details.

如果你想直接看 HTTP 接口、请求参数以及 SSE / 事件细节，请看 API 文档。
